Feature file: `docs/frontend_init.md`

## Goal
Stand up a dev-mode frontend in `frontend/` (Next.js App Router + TypeScript + Tailwind) that supports:
- signup + login (dev-mode; backend currently has no JWT auth endpoints)
- dashboard list of credit cases
- CRUD for customers + customer contacts
- credit case detail view (read/update)
- document uploads to customer or credit case

## Assumptions & constraints discovered
- Backend API base is `/api/v1/` (see `backend/core/config.py` and `backend/app/urls.py`).
- Backend currently exposes DRF ViewSets for:
  - `users`, `organizations`
  - `customers`, `customer-contacts`
  - `credit-cases`
  - `upload-documents`
- No JWT / login endpoints found in backend; all viewsets currently have no `permission_classes` and default auth is not configured in settings. Frontend login will therefore be **dev-only**: pick a user by email from `/users/` and store in localStorage.
- Credit case creation is likely not possible without auth because `CreditCaseSerializer` uses `CurrentUserDefault()` for `assigned_to`. Frontend will not rely on creating credit cases; it will support list + detail/update where feasible.

## Implementation plan
- Scaffold Next.js app in `frontend/` with pages:
  - `/signup`, `/login`
  - `/dashboard`
  - `/customers`, `/customers/new`, `/customers/[id]`
  - `/credit-cases/[id]`
- Add a small `lib/api.ts` wrapper for DRF (pagination, JSON + multipart uploads).
- Add a simple client-side auth store (`lib/auth.tsx`) backed by localStorage.
- Add shared UI primitives (buttons, inputs, cards) using Tailwind (skip shadcn CLI to avoid interactive prompts).
- Update `docker-compose.yml` to enable a `frontend` service that runs `npm run dev` and depends on `backend`.
- Add feature logs under `logs/implementations/` and `logs/errors/` as needed.
