Feature file: `docs/frontend_init.md`

## Summary of implementation
- Created a Next.js (App Router) frontend in `frontend/` with Tailwind styling.
- Implemented dev-mode auth:
  - `/signup` creates a backend user via `POST /api/v1/users/`
  - `/login` selects an existing user by email from `GET /api/v1/users/` and stores it in localStorage
- Implemented core pages:
  - `/dashboard`: lists credit cases from `GET /api/v1/credit-cases/` with a status filter
  - `/customers`: list customers
  - `/customers/new`: create customer and optional primary contact
  - `/customers/[id]`: view/update/delete customer, manage contacts, upload documents linked to customer
  - `/credit-cases/[id]`: view/update credit case request fields, upload documents linked to credit case
- Added a small DRF-friendly API wrapper in `frontend/src/lib/api.ts`:
  - JSON requests, multipart uploads, and “list all pages” helper for DRF pagination.
- Enabled a `frontend` service in `docker-compose.yml` that runs Next dev server on port `3000`.
- Updated backend CORS dev allowlist to include `http://localhost:3000` so browser API calls succeed.

## Notes / known backend limitations (dev mode)
- Backend currently has no JWT/session login endpoint configured; frontend login is dev-only.
- Creating credit cases via API may fail without auth because `assigned_to` uses `CurrentUserDefault()` in the backend serializer; frontend does not rely on credit-case creation.
